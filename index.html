Curso de GitHub

1. O que é Git e GitHub?

O Git é um sistema de controle de versão distribuído, criado por Linus Torvalds. Ele permite que desenvolvedores acompanhem e gerenciem alterações no código ao longo do tempo. Já o GitHub é uma plataforma que hospeda repositórios Git e oferece uma série de ferramentas para colaboração e gerenciamento de projetos.

2. Instalando o Git

Para começar a usar o Git, você precisa instalá-lo em seu computador. Siga os passos abaixo para instalar o Git:

  • Baixe o instalador do Git para o seu sistema operacional.
  • Execute o instalador e siga as instruções na tela.
  • Depois de instalado, abra o terminal ou prompt de comando e execute git --version para verificar se a instalação foi bem-sucedida.

3. Configurando sua conta no GitHub

Para usar o GitHub, você precisa criar uma conta. Siga os passos abaixo para configurar sua conta no GitHub:

  • Acesse a página de inscrição do GitHub.
  • Preencha o formulário de inscrição com seu nome, endereço de e-mail e senha.
  • Verifique seu endereço de e-mail e complete o processo de configuração da conta.

4. Criando um Repositório

Para começar a usar o Git e o GitHub, você precisa criar um repositório. Siga os passos abaixo para criar um repositório no GitHub:

  • Acesse sua conta no GitHub.
  • No canto superior direito, clique no ícone de + e selecione New repository.
  • Dê um nome ao seu repositório, adicione uma descrição opcional e escolha se o repositório será público ou privado.
  • Selecione Create repository.

5. Comandos Básicos do Git

Agora que você tem um repositório, vamos aprender alguns comandos básicos do Git:

  • git init: Inicializa um novo repositório Git.
  • git add .: Adiciona todos os arquivos modificados à área de preparação.
  • git commit -m "mensagem": Cria um commit com uma mensagem descritiva.
  • git push origin main: Envia as alterações para o repositório remoto.

6. Enviando Código para o GitHub

Depois de fazer alterações no seu código, você pode enviá-las para o GitHub. Siga os passos abaixo para enviar seu código:

  • Abra o terminal ou prompt de comando na pasta do seu repositório.
  • Adicione os arquivos modificados à área de preparação com git add ..
  • Crie um commit com uma mensagem descritiva usando git commit -m "mensagem".
  • Envie as alterações para o GitHub com git push origin main.

7. Colaboração no GitHub

O GitHub facilita a colaboração em projetos de software. Aqui estão algumas práticas recomendadas:

  • Use pull requests para revisar e discutir alterações antes de integrá-las ao projeto principal.
  • Crie issues para rastrear bugs e solicitar novos recursos.
  • Use branches para desenvolver novos recursos e corrigir bugs sem afetar o código principal.

8. GitHub Desktop

O GitHub Desktop é uma interface gráfica que simplifica o uso do Git e do GitHub, permitindo gerenciar repositórios, branches e commits de maneira intuitiva. Aqui estão alguns passos para começar com o GitHub Desktop:

  • Instalação do GitHub Desktop:
    • Visite o site oficial do GitHub Desktop e baixe o instalador para o seu sistema operacional.
    • Execute o instalador baixado e siga as instruções na tela para completar a instalação.
  • Configuração Inicial:
    • Após a instalação, abra o GitHub Desktop.
    • Faça login na sua conta do GitHub ou crie uma nova conta diretamente pelo aplicativo.
  • Clonando um Repositório:
    • No GitHub Desktop, clique em "File" (Arquivo) e selecione "Clone repository" (Clonar repositório).
    • Escolha o repositório que você deseja clonar da lista de repositórios disponíveis na sua conta do GitHub.
    • Escolha o local onde deseja salvar o repositório clonado no seu computador.
  • Realizando Commits:
    • Após fazer alterações nos arquivos do seu projeto, o GitHub Desktop mostrará essas alterações.
    • Adicione uma descrição para o commit na caixa de texto abaixo da lista de arquivos alterados.
    • Clique em "Commit to main" (Commit para a branch principal) para salvar as alterações localmente.
  • Sincronizando com o GitHub:
    • Para enviar suas alterações para o GitHub, clique em "Push origin" (Enviar para o GitHub).
    • Isso sincroniza as alterações locais com o repositório remoto no GitHub.
  • Gerenciando Branches:
    • Clique no menu "Current branch" (Branch atual) para criar novas branches, alternar entre branches existentes ou excluir branches que não são mais necessárias.
  • Recebendo Atualizações:
    • Para obter as últimas alterações de um repositório remoto, clique em "Fetch origin" (Buscar do GitHub) para atualizar sua cópia local do repositório.
  • Visualização de Histórico:
    • Utilize a guia "History" (Histórico) para visualizar todos os commits feitos no repositório, incluindo mensagens de commit e alterações associadas.
